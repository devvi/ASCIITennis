# Issue 5 — Task Analysis: Web Port

## Related Modules

- `src/main.lua` — TIC-80 entry point, `TIC()` callback, `cls()`, `print()` → replace with HTML/JS
- `src/input.lua` — `btn()` → replace with `keydown`/`keyup` and `click` events
- `src/render.lua` — `print()` → replace with Canvas `fillText()`
- `src/camera.lua` — `print()` at pixel coords → Canvas text rendering
- All other modules — game logic (pure Lua, kept as-is or transpiled)

## Impacts

- **Platform shift**: TIC-80 Lua → browser JavaScript (or fengari transpiled Lua)
- **Input overhaul**: button IDs → WASD + mouse click with combo detection
- **Rendering replacement**: `cls()`/`print()` → Canvas 2D text API with monospace font
- **Game loop**: `TIC()` callback → `requestAnimationFrame()` with delta time
- **Shot combo logic**: mouse + WASD to replace B+arrow combos (flat, topspin, slice, lob)

## Summary

Port the existing Lua game to a standalone web page (HTML + CSS + JS). Game logic (physics, AI, scoring, court, player, ball) can be translated from Lua to JavaScript with minimal changes. The rendering, input, and platform layers need complete replacement. Output is a single `index.html` file with embedded JS or separate files under `src/` with a JS entry point.

## Refined Implementation Phases

### Phase 1: Project Scaffold
- Create `index.html` with Canvas element, monospace font CSS, and `<script>` tags loading JS files in dependency order (NOT `<script type="module">` — ES modules are blocked on `file://` protocol)
- Add Canvas element with 960×540 logical resolution (4× scale of TIC-80 240×136)
- Set monospace font CSS (`"Courier New", monospace`)
- Create stub JS files: `constants.js`, `court.js`, `camera.js`, `player.js`, `ball.js`, `ai.js`, `scoring.js`, `input.js`, `render.js`, `main.js`
- Fix CORS bug: ES modules (`<script type="module">`) are blocked by browsers on `file://` URLs. Solution: use regular `<script>` tags with global objects. JS files loaded in order via `<script src="...">` (no `type="module"`). Each file creates/extends a global `game` namespace object.
- Add `start.sh` / `package.json` with local dev server command (`python3 -m http.server` or `npx serve`) for users who prefer HTTP serving

### Phase 2: Port Constants & Game Logic Modules
- **constants.js**: All court dims, physics, hit types, button mappings, state enums, AI configs
- **court.js**: `init()`, `is_in_bounds()`, `is_in_player_side()`, `hits_net()` — pure translation
- **player.js**: `new()`, `update()`, `move()`, `swing()`, `can_hit()` — pure translation
- **ball.js**: `new()`, `update()`, `serve()`, `hit()` — pure translation
- **ai.js**: `new_player()`, `update()` — pure translation
- **scoring.js**: `new()`, `point_name()`, `display()`, `award_point()`, `award_game()`, `award_set()` — pure translation
- **camera.js**: `init()`, `project()`, `project_char()`, `draw_line()`, `draw_rect()` — uses Canvas context instead of `print()`

### Phase 3: Input Module
- `input.js`: `init()`, `update()`, `pressed()`, `held()`, `released()`, `get_movement()`, `get_shot_type()`, `get_serve()`
- `keydown`/`keyup` listeners on `window` for W/A/S/D → map to UP/DOWN/LEFT/RIGHT
- `mousedown`/`mouseup` listeners on Canvas for BTN_B (and BTN_A)
- Virtual button array (curr/prev) pattern preserved from Lua

### Phase 4: Render Module
- `render.js`: `court()`, `net()`, `player()`, `ball()`, `hud()`, `menu()`, `game_over()`
- Canvas rendering: `ctx.clearRect()` → `cls()`, `ctx.fillText()` → `print()`
- Character-cell coordinate system: logical 240×136 grid, scaled 4× for canvas pixels
- `camera.draw_line()` draws depth-shaded characters via `fillText()`
- `camera.project_char()` maps 3D → (sx, sy, char) using same projection math

### Phase 5: Main Game Loop
- `main.js`: `init_game()`, `start_match()`, `setup_serve()`, `do_serve()`, `resolve_point()`
- State machine: `update_menu()`, `update_serving()`, `update_playing()`, `update_point_scored()`, `update_game_over()`
- Game loop: `requestAnimationFrame()` → `input.update()` → state update → `draw_game()`
- Same game logic as Lua `main.lua` with TIC-80 APIs removed
