# Issue 1 — Task Analysis

## Related Modules

- `src/main.lua` — TIC-80 entry point, game loop, input handling, screen drawing
- `src/player.lua` — Player position, movement, racket state, hit logic
- `src/ball.lua` — Ball physics, trajectory, speed, spin
- `src/ai.lua` — AI opponent behavior, difficulty levels, decision making
- `src/court.lua` — Court layout, net, lines, boundaries in ASCII
- `src/scoring.lua` — Tennis scoring logic (points, games, sets, match)
- `src/render.lua` — ASCII 3D projection, sprites, HUD rendering
- `src/input.lua` — Button mapping and state management

## Impacts

- **TIC-80 platform constraints** — code must fit within TIC-80's limits (64KB code, 256 sprites, 4 banks of 256x256 pixel screen)
- **ASCII rendering engine** — needs a 3D-to-ASCII projection system for PS1-style perspective
- **Ball physics** — trajectory calculation with gravity, bounce, and spin effects
- **AI system** — must be configurable by difficulty; decides positioning and shot selection
- **Scoring state machine** — tracks point/game/set/match state with tennis rules
- **Input mapping** — directional movement + action button with combination detection for shot types

## Summary

This is a greenfield TIC-80 game. All modules need to be built from scratch. The core challenge is rendering a convincing 3D tennis experience in ASCII on a constrained platform, with responsive controls, ball physics, and competent AI.

## Implementation Phases

### Phase 1: Project Setup, Data Structures, and State Machine
- Create `src/` directory with empty module stubs
- Define all constants in `src/constants.lua` (court dimensions, physics params, key mappings)
- Define all data structures (player, ball, AI config, score)
- Implement game state machine skeleton in `src/main.lua` (MENU, SERVING, PLAYING, POINT_SCORED, GAME_OVER)

### Phase 2: Court and 3D ASCII Projection
- Implement perspective projection in `src/camera.lua`
- Render court lines, net, and boundaries as ASCII in `src/court.lua`
- Depth shading using character density ` .:-=+*#%@`

### Phase 3: Player Movement and Input
- Implement button reading and combo detection in `src/input.lua`
- Implement player movement (left/right/forward/backward) in `src/player.lua`
- Connect input to player movement in game loop

### Phase 4: Ball Physics and Hit Mechanics
- Implement ball trajectory with gravity and bounce in `src/ball.lua`
- Implement hit types (flat, top spin, slice, lob) with button combos
- Implement serve logic
- Collision detection with court boundaries and net

### Phase 5: AI Opponent
- Implement AI decision-making in `src/ai.lua`
- Two difficulty levels: easy and hard
- AI positions itself relative to ball trajectory and selects shot type
- Configurable reaction time, accuracy, aggression, speed

### Phase 6: Scoring Logic
- Implement tennis scoring (15, 30, 40, deuce, advantage, game) in `src/scoring.lua`
- Game and set tracking
- Match win condition

### Phase 7: Game Flow and Menu
- Title screen with rival difficulty selection
- Serve → rally → point resolution loop
- Score HUD display
- Game over screen with restart option

### Phase 8: Testing and Polish
- Verify state machine transitions
- Test all hit types and scoring edge cases
- Balance AI difficulty levels
- Ensure code fits TIC-80 constraints
