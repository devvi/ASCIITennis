# Issue 1 — ASCIITennis Design

## Architecture Overview

```
TIC-60 (60 FPS)
  │
  ├─ main.lua (global state, game FSM, TIC() entry point)
  │   ├─ input.lua     → read & debounce buttons
  │   ├─ player.lua    → movement, racket, hit types
  │   ├─ ai.lua        → opponent decision-making
  │   ├─ ball.lua      → physics (gravity, bounce, spin)
  │   ├─ court.lua     → court geometry & collision
  │   ├─ scoring.lua   → tennis rules (points/games/sets)
  │   └─ render.lua    → 3D ASCII projection → screen
  └─── cart data (sprites, fonts, maps)
```

## Data Structures

```lua
-- player.lua
player = {
  x = 0,           -- horizontal position on court
  z = -40,         -- depth (negative = player's side)
  swing_timer = 0, -- frames remaining in swing animation
  hit_type = nil,  -- nil | "flat" | "topspin" | "slice" | "lob"
  facing = 1       -- 1 = toward net, -1 = retreating
}

-- ball.lua
ball = {
  x = 0, y = 2, z = 0,   -- world position (y = height above court)
  vx = 0, vy = 0, vz = 0, -- velocity per frame
  spin = 0,                -- topspin/backspin modifier
  state = "serve"          -- "serve" | "rally" | "dead"
}

-- ai.lua
ai = {
  x = 0,  z = 40,    -- opponent position (far side)
  diff = "medium",    -- "easy" | "medium" | "hard"
  reaction = 10,      -- frames of reaction delay
  target_x = 0,       -- where AI is moving toward
  target_z = 0
}

-- court.lua (constants)
court = {
  width = 20,           -- half-width in world units
  length = 80,          -- half-length (net at z=0)
  net_height = 3,       -- net height in world units
  server_x = { left = -4, right = 4 }
}

-- scoring.lua
scoring = {
  state = "serve",       -- "serve" | "rally" | "point" | "game_over"
  server = 0,            -- 0 = player, 1 = AI
  points = { 0, 0 },     -- 0,15,30,40,A stored as {0,1,2,3,4}
  games = { 0, 0 },      -- games won this set
  sets = { 0, 0 },       -- sets won this match
  display_points = { "", "" }
}

-- Game state machine (main.lua)
state = "TITLE"  -- TITLE | SELECT | SERVE | PLAY | POINT | GAME | MATCH_END
```

## Game State Machine

```
TITLE ──► SELECT ──► SERVE ──► PLAY ──► POINT ──► SERVE
                  │                   │               │
                  │                   └─► GAME ──► SERVE
                  │                               │
                  │                               └─► MATCH_END
                  └──────────────────────────────────────┘
```

## Module Design

### Phase 1 — Data & Constants
- `court.lua`: court dimensions, net height, service box positions
- `render.lua`: character palette, projection constants
- `scoring.lua`: scoring translation table

### Phase 2 — Core Logic
- `input.lua`: TIC-80 button() wrapper with combo detection
- `player.lua`: position clamping, swing state, hit type from button combos
- `ball.lua`: gravity, bounce off court/net/racket, spin curve
- `ai.lua`: target selection, movement, shot choice by difficulty
- `scoring.lua`: state machine for serve→rally→point, tennis rules (deuce, advantage)

### Phase 3 — UI / Rendering
- `render.lua`: 3D perspective projection, draw court lines, net, player/ball sprites, HUD
- Screen layout: ASCII art court with PS1-style jitter/wobble, score overlay at top

### Phase 4 — Integration & Polish
- `main.lua`: game loop, state transitions, menu screen
- Tie all modules together, final TIC() function
- Difficulty select screen, title screen

## 3D ASCII Rendering Approach

- World space: x (left-right), y (up), z (depth, net at z=0)
- Projection: simple perspective divide `screen_x = focal_length * x / z`, `screen_y = focal_length * y / z`
- ASCII characters mapped by depth: `" .:-=+*#%@"` for distance shading
- Ball rendered as `o`, net as `|||`, player/AI as stick figures built from chars

## Tennis Scoring Rules

- Points: 0→15→30→40→Game (4-point lead with ≥2 gap)
- Deuce at 40-40; advantage after winning a point at deuce
- Game: first to 6 games with 2-game lead (tiebreak at 6-6)
- Match: best-of-1 set (single set wins match)
