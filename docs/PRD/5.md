# Issue 5 — Port ASCIITennis from TIC-80 to Web Platform

## Motivation

TIC-80 platform testing overhead is too high. Porting to web enables faster development, easier testing, and broader accessibility.

## Features

- Same ASCII tennis game running in a web browser
- WASD keyboard controls (W=up, A=left, S=down, D=right)
- Left mouse button for action (hit, serve, confirm)
- HTML5 Canvas-based rendering of ASCII graphics
- Game logic (physics, AI, scoring) preserved as-is

## Acceptance Criteria

- [ ] Game loads and runs in a modern web browser (Chrome, Firefox, Edge)
- [ ] Player moves with W/A/S/D keys (previously arrow keys)
- [ ] Player hits ball, serves, and confirms selections with left mouse click (previously B button)
- [ ] All 4 shot types work (flat, topspin, slice, lob) using mouse + movement keys
- [ ] Court, net, players, ball, HUD, and menus rendered as ASCII on canvas
- [ ] Game loop runs at 60fps via requestAnimationFrame
- [ ] AI opponent, scoring, state machine unchanged
- [ ] All TIC-80 APIs (TIC, btn, cls, print) eliminated
