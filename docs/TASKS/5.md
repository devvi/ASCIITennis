# Issue 5 — Task Analysis: Web Port

## Related Modules

- `src/main.lua` — TIC-80 entry point, `TIC()` callback, `cls()`, `print()` → replace with HTML/JS
- `src/input.lua` — `btn()` → replace with `keydown`/`keyup` and `click` events
- `src/render.lua` — `print()` → replace with Canvas `fillText()`
- `src/camera.lua` — `print()` at pixel coords → Canvas text rendering
- All other modules — game logic (pure Lua, kept as-is or transpiled)

## Impacts

- **Platform shift**: TIC-80 Lua → browser JavaScript (or fengari transpiled Lua)
- **Input overhaul**: button IDs → WASD + mouse click with combo detection
- **Rendering replacement**: `cls()`/`print()` → Canvas 2D text API with monospace font
- **Game loop**: `TIC()` callback → `requestAnimationFrame()` with delta time
- **Shot combo logic**: mouse + WASD to replace B+arrow combos (flat, topspin, slice, lob)

## Summary

Port the existing Lua game to a standalone web page (HTML + CSS + JS). Game logic (physics, AI, scoring, court, player, ball) can be translated from Lua to JavaScript with minimal changes. The rendering, input, and platform layers need complete replacement. Output is a single `index.html` file with embedded JS or separate files under `src/` with a JS entry point.

## Implementation Phases

### Phase 1: Project Setup and Build Infrastructure
- Create `index.html` with Canvas element and monospace font styling
- Set up JS module structure mirroring Lua modules
- Port constants to JS

### Phase 2: Keyboard and Mouse Input
- Implement `keydown`/`keyup` event listeners for W/A/S/D → direction
- Implement `click`/`mousedown` event for action button
- Map input to virtual button states to preserve existing combo logic
- Shot combos: click + WASD direction → flat/topspin/slice/lob

### Phase 3: Canvas Renderer
- Replace `cls()` with `ctx.clearRect()`
- Replace `print(str, x, y)` with `ctx.fillText()` using monospace font at fixed cell size
- Implement depth-shaded ASCII character rendering matching TIC-80 look
- Scale factor for readability (e.g., 8x per pixel or character cell size)

### Phase 4: Game Loop
- Replace `TIC()` with `requestAnimationFrame()` loop
- Port state machine, `init_game()`, `draw_game()`, `update_game()` to JS
- Maintain 60fps timing

### Phase 5: Translate Game Logic Modules
- Port `court.lua`, `camera.lua`, `player.lua`, `ball.lua`, `ai.lua`, `scoring.lua` to JS
- Preserve all game mechanics, physics, and scoring rules

### Phase 6: Polish
- Mouse cursor lock or click-to-focus for better UX
- Keyboard shortcut hints on screen
- Verify all game states: menu, serving, playing, point scored, game over
- Test all 4 shot types via mouse + WASD combos
