# Issue 1 — ASCIITennis Design

## Architecture

TIC-80 Lua game with a fixed 60fps game loop via the `TIC()` callback.

### State Machine

```
MENU → SERVING → PLAYING → POINT_SCORED → SERVING
                          ↘ GAME_OVER
```

- **MENU**: Title screen, rival difficulty selection
- **SERVING**: Serve setup, ball placed, server tosses
- **PLAYING**: Active rally, player/AI move and hit
- **POINT_SCORED**: Brief pause, score update display
- **GAME_OVER**: Match result, option to restart

### Module Design

| Module | Responsibility |
|---|---|
| `src/main.lua` | TIC() entry, state machine dispatch, load other modules |
| `src/constants.lua` | Court dimensions, physics constants, key mappings, ASCII art |
| `src/input.lua` | Read TIC-80 buttons, detect combos for shot types |
| `src/court.lua` | 3D court coordinates, net, lines, boundary checks |
| `src/camera.lua` | First-person 3D-to-2D ASCII projection (perspective) |
| `src/player.lua` | Player position, movement, racket state, hit timing |
| `src/ball.lua` | Ball position/velocity, gravity, bounce, spin, collision |
| `src/ai.lua` | AI decision-making, difficulty config, target position, shot selection |
| `src/scoring.lua` | Points/games/sets state machine, tennis rules (deuce, advantage) |
| `src/render.lua` | ASCII drawing primitives, 3D projection render, HUD overlay |

### Data Structures

```lua
-- Player (human or AI)
player = {
  x, z,           -- position on court (camera is at player eyes)
  state,          -- "idle", "moving", "hitting"
  hit_timer,      -- cooldown after swing
  facing          -- direction angle
}

-- Ball
ball = {
  x, y, z,        -- 3D position
  vx, vy, vz,     -- velocity components
  spin_x, spin_z, -- spin affecting trajectory
  state           -- "held", "in_play", "out", "net", "bounce"
}

-- AI config per difficulty
ai_config = {
  reaction_time,  -- frames before AI responds
  accuracy,       -- how close to ideal target
  aggression,     -- tendency to hit hard/approach net
  speed           -- movement speed multiplier
}

-- Score
score = {
  points = {0, 0},       -- 0, 1, 15, 30, 40...
  games = {0, 0},
  sets = {0, 0},
  deuce = false,
  advantage = nil,       -- nil | 0 | 1
  state = "playing"      -- "playing" | "game" | "set" | "match"
}
```

### 3D ASCII Projection

Simple perspective projection from a camera at (player.x, player_eye_y, player.z):
- Project 3D world points onto a 2D screen plane using: `screen_x = fov * (world_x - cam_x) / (world_z - cam_z)`, `screen_y = fov * (world_y - cam_y) / (world_z - cam_z)`
- Map screen coordinates to TIC-80's 240×136 resolution
- Render court lines, net, ball, and players as ASCII characters with PS1-style affine texture mapping
- Use character selection for depth shading (e.g. ` .:-=+*#%@` for distance)

### Hit Types (Button Combos)

- **Flat** (B only): Fast straight shot
- **Top Spin** (B + Up): Arc over net, dips fast
- **Slice** (B + Down): Low trajectory, skids on bounce
- **Lob** (B + Left/Right): High arc over opponent's head
